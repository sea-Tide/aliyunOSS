1、引入依赖到`pom.xml`文件中：

```xml
	<repositories>
		<repository>
		    <id>jitpack.io</id>
		    <url>https://jitpack.io</url>
		</repository>
	</repositories
```

```xml
	<dependency>
	    <groupId>com.github.sea-Tide</groupId>
	    <artifactId>aliyunOSS</artifactId>
	    <version>v1.0.0</version>
	</dependency>
```

2、到阿里云注册OSS对象存储服务：https://oss.console.aliyun.com/overview
①、点击bucket列表，创建bucket，输入bucket名称后直接创建
②、鼠标移动到右上角的用户信息中，会看到AccessKey，点击进入，点击“使用RAM用户AccessKey”，进来后点击“身份管理”，在“用户”模块中点击创建用户，输入名称后，勾选“使用永久AccessKey访问”，点击确定，复制记录这边的accessKeyId和accessKeySecret，只会出现这一次，没记录就需要重新创建账户
③、回到bucket列表，点击创建的bucket名称后，会看到“权限控制”模块，设置权限阻止公网访问关闭，公共读权限，bucket授权策略中新增授权子用户RAM账号

3、在你的yaml文件中配置以下信息：

```yaml
aliyun:
  oss:
    endpoint: https://oss-cn-hangzhou.aliyuncs.com  # 点击bucket名称后到“概述”最下方复制，与我保持一致也可以，一般没修改都是默认hangzhou
    bucketName: sh-web-tlias-project # 你刚刚创建的bucket名称
    accessKeyId: yourRAMaccessKeyId # 你RAM账号的accessKeyId，如果报错可以将Id和Secret配置到用户环境变量中，用环境变量引入(官方推荐)
    accessKeySecret: yourRAMaccessKeySecret # 你RAM账号的accessKeySecret
```

4、在你的`controller`层中使用`@Autowired`注解依赖注入`AliOSSUtils`这个工具类，使其成为`IOC`外部容器的`bean`对象：

```java
@Autowired
private AliOSSUtils aliOSSUtils;
```

5、类似我这里的使用aliOSSUtils.upload()方法，传递你前端页面发送request请求中携带的文件数据，后端使用MultipartFile对象来接收文件类型，返回url给前端数据回显，前端即可展示图片

```java
@RestController
public class UploadController {
    @Autowired
    private AliOSSUtils aliOSSUtils;
    
    @PostMapping("/upload")
    public String upload(MultipartFile image) throws IOException {
        String url = aliOSSUtils.upload(image);
        return url;
    }
}
```
